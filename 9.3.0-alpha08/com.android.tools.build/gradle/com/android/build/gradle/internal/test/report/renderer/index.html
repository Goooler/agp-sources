Introduction to Test Data

This report provides the results of your test execution. It summarizes the pass/fail status of your tests, helping you identify regressions and ensure the quality of your application.

Navigating & Filtering
Search & Filter

Searching filters the table while preserving the project hierarchy. It keeps matching items and their parent structures visible, allowing you to see exactly which modules or packages contain the results.

Use Add filter to narrow down the data by specific Modules, Packages, Classes, Test Cases, or Status. You can also filter the entire report by test suite or build variant.

View Modes & Grouping
Switch between Flat View (list of all items) and Hierarchical View (nested folders). In Flat View, use Group By to aggregate data by module, package, or class.
Sorting
Click on the first column header (e.g., Module, Package, Class) to sort the table alphabetically by those names.
Interactive Features
Debug Failures
Clicking on the name of a failed test case in the report table opens its Stack Trace. Multiple failures across suites or variants will be shown side-by-side.
Accessibility Features
  • Skip Links: Keyboard users can use the "Skip to main content" link (press Tab after page load) to jump directly to the data table.
  • Column Resizing: Focus a column border resizer and use Arrow Keys to adjust width, or Enter to reset. Double-click with a mouse also resets width.
  • Screen Reader Announcements: All view changes and dynamic filter updates are announced via an aria-live region.
Legend
Symbols & Colors

-: Indicates that no data is available for that variant.

Status & Rate Colors:

  • Green: Passed tests or high pass rate (95% or above)
  • Yellow: Skipped tests or medium pass rate (80% to 94.9%)
  • Red: Failed tests or low pass rate (below 80%)

Note on Pass Rate: The pass rate is calculated as passed / (passed + failed). Skipped tests are excluded from this calculation.

Passed Tests
Tests that completed successfully without any assertions failing or unexpected exceptions.
Failed Tests
Tests where an assertion failed or an unexpected exception was thrown during execution.
Skipped Tests
Tests that were ignored or disabled, often using annotations like @Ignore.
