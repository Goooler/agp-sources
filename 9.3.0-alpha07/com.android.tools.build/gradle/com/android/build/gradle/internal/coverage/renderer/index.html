Introduction to Coverage Data

This report provides code coverage information for your project, showing how much of your code was executed during testing. Higher percentages indicate better coverage.

Navigating & Filtering
Search & Filter

Searching filters the table while preserving the project hierarchy. It keeps matching items and their parent structures visible, allowing you to see exactly which modules or packages contain the results.

Use Add filter to narrow down the data by specific Modules, Packages, or Classes. You can also filter the entire report by test suite or build variant.

View Modes & Grouping
Switch between Flat View (list of all items) and Hierarchical View (nested folders). In Flat View, use Group By to aggregate data by module, package, or class.
Sorting
Click on column headers or subheaders (e.g., Instruction, Branch) to sort the table by those values.
Interactive Features
Source View Deep Dive
Clicking on a Class Name will open a detailed source view, showing line-by-line coverage directly on your code.
Table Legend
Symbols & Colors

--: Indicates that the metric is not applicable (e.g., no instructions or branches to cover in that element).

Coverage Percentage Colors:

  • Green: High coverage (80% or above)
  • Yellow: Medium coverage (60% to 79.9%)
  • Red: Low coverage (below 60%)
  • Gray: Not applicable
Instruction Coverage
Percentage of Java bytecode instructions that were executed.
Branch Coverage
Percentage of control flow branches (e.g., if/else) that were executed.
